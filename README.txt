【重要提示】
1. 声明：本开发包代码只是为了方便机构\商户测试而提供的样例代码，机构\商户可以根据自己需要，按照技术文档编写。本代码仅供参考；
2. 联调前请仔细阅读以下说明；
3. 测试时一定要注意配置好配置文件中的各种证书文件的路径；
4. 注意使用国密必须更新bc的jar（lib目录中已包含）
5. 使用低于1.8.0_161版本的JDK可能会导致敏感信息解密失败问题

关于证书配置
1. 机构(或商户)国密签名证书(后缀为sm2)配在【uassdk.sm2.signCert.path】中（绝对路径精确到文件名），对应的密码配在【uassdk.sm2.signCert.pwd】
2. 机构(或商户)国密加密证书（后缀为cer）配在【uassdk.sm2.encryptCert.path】中（绝对路径精确到文件名）
3. 机构(或商户)国密验证银联签名证书（后缀为cer）配在【uassdk.sm2.validateCert.dir】（绝对路径精确到文件所在目录级别）
4. 机构(或商户)国密解密银联加密报文（后缀为sm2）配在【uassdk.sm2.decryptCert.path】中（绝对路径精确到文件名），对应的密码配在【uassdk.sm2.decryptCert.pwd】
5. 机构(或商户)国际签名证书(后缀为pfx)配在【uassdk.signCert.path】中（绝对路径精确到文件名），对应的密码配在【uassdk.signCert.pwd】
6. 机构(或商户)国际加密证书（后缀为cer）配在【uassdk.encryptCert.path】中（绝对路径精确到文件名）
7. 机构(或商户)国际验证银联签名证书（后缀为cer）配在【uassdk.validateCert.dir】（绝对路径精确到文件所在目录级别）
8. 机构(或商户)国际解密银联加密报文（后缀为sm2）配在【uassdk.decryptCert.path】中（绝对路径精确到文件名），对应的密码配在【uassdk.decryptCert.pwd】
